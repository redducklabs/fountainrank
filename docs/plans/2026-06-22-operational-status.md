## `app/conditions.py` (precise algorithm — resolves plan-review-1 MAJORs)
- Category map: `working→ok`; `{broken,blocked}→not_working`; everything else `→degraded`. Severity rank `not_working(3) > degraded(2) > ok(1)`.
- `@dataclass(frozen=True) StatusResult(current_status: str|None, last_verified_at: datetime|None)`.
- `derive_status(reports: list[tuple[str, uuid.UUID, datetime]], *, now, freshness_days, corroboration_min) -> StatusResult` — pure. `reports` = (status, user_id, created_at) of **non-hidden** rows.

  **Corroboration-timestamp model** (Codex-endorsed in plan-review-2 — report-level evidence with an Nth-most-recent-distinct-user corroboration timestamp; single actor can't flip in either direction):
  1. `last_verified_at` = max `created_at` among ALL `working` reports (not window-bound).
  2. In-window = `created_at >= now - freshness_days`. If none → `StatusResult(None, last_verified_at)`.
  3. For each category, gather its in-window reports and, **per distinct user, that user's most-recent report IN THAT category** → a set of per-user latest-in-category timestamps. (A user who reported broken then later working appears in BOTH `not_working` and `ok` — their broken evidence is NOT erased, so a single recant can't dissolve a corroborated outage.)
  4. A category is **corroborated** iff it has ≥ `corroboration_min` distinct users. Its **corroboration timestamp** = the `corroboration_min`-th most-recent of those per-user timestamps (the 2nd-most-recent for the default 2) — the moment it last had `corroboration_min` distinct fresh supporters.
  5. `current_status` = the corroborated category with the **greatest corroboration timestamp**, tie-broken by **severity** (`not_working(3) > degraded(2) > ok(1)`). Recovery therefore needs ≥2 distinct users whose `working` corroboration timestamp is newer than the outage's — one new + one stale working, or two from the same user, leave the corroborated outage in place; one later re-report by an existing supporter does not raise a category's corroboration timestamp.
  6. If no category is corroborated: `reported_issue` if any user's **latest-overall** in-window report is `not_working`/`degraded` (a current, uncorroborated issue — a recanted broken does not count), else `NULL` (fall back to baseline `is_working`).
- `async recompute_fountain_status(session, fountain_id, *, now=None)` — read non-hidden reports for the fountain, `derive_status(now=now or datetime.now(UTC), ...)`, set `Fountain.current_status`/`last_verified_at`, flush. Called from the write path AND any future moderation hide/unhide. Logs `fountain_id`, old→new `current_status` at DEBUG.

## API
- `POST /api/v1/fountains/{id}/conditions` (auth): body `ConditionReportRequest{ status: Literal[...8...], is_proximate: bool = False }` (Pydantic enum → 422). Load fountain `FOR UPDATE` (is_hidden filter, 404). **Capture one `report_time = datetime.now(UTC)`** used for the inserted row's `created_at`, the `recompute_fountain_status(now=report_time)`, AND the dedup day — so the report timestamp and dedup day can't straddle a UTC-midnight boundary (plan-review-1 MINOR). Insert report (`RETURNING id`). `recompute_fountain_status`. Emit one event: `verify_working` (3) if `status=='working'` else `report_condition` (2); `target_type='condition_report'`, `target_id`=report id; dedup per day (`verify:{u}:{f}:{yyyymmdd}` / `cond:{u}:{f}:{yyyymmdd}`, day from `report_time`); `location` = `point_geography` from queried coords. Commit; return `FountainDetail`.
- **Structured logging (CLAUDE.md):** the endpoint logs at INFO `fountain_id`, `user_id`, `report_id`, `status`, old→new `current_status`, and inserted-vs-deduped event — non-sensitive context, no PII/tokens.
- `FountainDetail` + `FountainPin` gain `current_status: str|None`, `last_verified_at: datetime|None`. `serialize_fountain_detail`, `nearby_fountains`, `fountains_in_bbox` select + populate them (additive — existing clients ignore).

## Tests (key cases)
- `derive_status` (covers every plan-review-1 + review-2 case): single working → `last_verified_at` set, `current_status=NULL` (uncorroborated → baseline); single broken → `reported_issue`; 2 distinct broken → `not_working`; **2 distinct broken then 2 distinct newer working → `ok`** (recovery); **2 distinct broken, then ONE of those same users reports working → stays `not_working`, `last_verified_at` updated** (review-2 sharp case — single actor can't clear); **2 broken + 2 newer working from the SAME user → stays `not_working`**; **2 broken + 1 newer working + 1 OLDER working (distinct users) → stays `not_working`** (recovery needs 2 distinct *fresh*); single user broken then working (no corroboration) → `NULL` (latest-overall working → no current issue); **degraded corroborated 09:00/09:01 + not_working corroborated 10:00/10:01 + an existing dirty reporter re-reports 10:02 → stays `not_working`** (corroboration-timestamp, not category-max-recency); equal-timestamp degraded vs not_working → severity → `not_working`; out-of-window ignored; hidden excluded (recompute DB test with a seeded fountain + users).
- API: report changes `current_status` only with corroboration; `verify_working` sets `last_verified_at`; detail + a bbox pin expose the fields; auth required (401); 422 on bad status; event emitted with target linkage; per-day dedup (second same-day verify awards 0).
